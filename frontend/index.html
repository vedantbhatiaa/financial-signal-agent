How it works
Three layers, one answer.

Every response is grounded in data you own — not LLM training memory.

01
Ingest
Price data, live news, and SEC filings are continuously pulled from three heterogeneous sources and stored in PostgreSQL, MongoDB, and a vector database — each optimised for its data type.
POSTGRESQL · MONGODB · CHROMADB
02
Embed
News articles and regulatory filings are chunked and converted to 384-dimension semantic vectors using a local embedding model. Retrieval is by meaning, not keywords.
RAG · ALL-MINILM-L6-V2
03
Reason
A Llama 3 agent selects the right tools — SQL for price queries, semantic search for sentiment, MongoDB for filing metadata — and synthesises a cited, grounded answer.
LANGCHAIN · GROQ · LLAMA 3
